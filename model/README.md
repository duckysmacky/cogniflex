# Рабочая область моделей

Эта директория предназначена для исследований, обучения, оценки моделей и
сохранения произведенных артефактов.

Production-код инференса находится в `ml-service/`. ML-сервис не должен
импортировать код из `model/training`; если шаг предобработки нужен в runtime,
минимальную runtime-версию нужно перенести или продублировать в `ml-service/app`.

## Структура

- `notebooks/` - только исследовательские и экспериментальные ноутбуки.
- `training/` - переиспользуемые утилиты для обучения и работы с датасетами.
- `artifacts/` - локально произведенные артефакты моделей, например `.pth`.
- `requirements.txt` - зависимости для исследований и обучения моделей.

Крупные файлы артефактов намеренно не коммитятся.
